## рҹҺҜ Quick Reference Table

### Decision Points

| Question | If YES | If NO |
|----------|--------|-------|
| Has bonus tokens added? | Go to "With Bonus" | Go to "No Bonus" |
| Current balance >= bought? | Scenario 1 (100%) | Check balance < bought |
| All current tokens used? | Check purchased tokens | Scenario 5/6 |
| Purchased tokens all intact? | Scenario 4 (calc bonus) | Scenario 5/6 |
| Effective tokens >= 1? | Scenario 5B (calc %) | Scenario 5A (not eligible) |

## рҹ’° Refund Amount Calculations

### No Bonus Path
```
If Balance >= Bought:
в”ңв”Җ Result: Scenario 1 вң“
в””в”Җ Refund: 100%

If Balance < Bought:
в”ңв”Җ Result: Scenario 2 вң“
в”ңв”Җ Formula: (Paid Г· Bought) Г— Current
в””в”Җ Example: (вӮ№1000 Г· 100) Г— 30 = вӮ№300
```

### With Bonus Path
```
If All Unused:
в”ңв”Җ Result: Scenario 3 вң“
в””в”Җ Refund: 100%

If Some Bonus Used:
в”ңв”Җ Result: Scenario 4 вң“
в”ңв”Җ Bonus Used = Added - Current
в”ңв”Җ Effective = Bought - Bonus Used
в”ңв”Җ Formula: (Paid Г· Bought) Г— Effective
в””в”Җ Example: (вӮ№1000 Г· 100) Г— 70 = вӮ№700

If Both Used:
в”ңв”Җ Effective = Current - (Added - Current)
в”ңв”Җ If Effective < 1: Scenario 5A (Not Eligible)
в”ңв”Җ If Effective >= 1: Scenario 5B (Partial)
в””в”Җ Formula: (Paid Г· Bought) Г— Effective
```

4. **Refund Cap**: Never exceeds the amount originally paid
5. **7-Day Window**: Refund requests only valid within 7 days (admin checks)
